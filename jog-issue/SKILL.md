# jog-issue: jog に draft Issue を作成する

ユーザーの入力を受け取り、構造化された Issue を draft ステータスで作成する。

---

## ステップ 0: jog ツールのロード

jog MCP ツールは遅延ロードなので、最初に ToolSearch で検索する。

```
ToolSearch: "jog issue"
```

---

## ステップ 1: 入力を受け取る

ユーザーの入力は3つのパターンがある：

**パターン A: タイトルのみ（短文）**
- そのままタイトルとして使う
- body は「（詳細未記入）」で作成し、あとで追記できる旨を伝える

**パターン B: 自然言語での説明**
- タイトルと body を自分で整形して構造化する
- body には「目的・背景」「やること（スコープ）」「受け入れ条件」を含める
- 不明な部分は TODO として記載する

**パターン C: 既存テキスト（会話ログ・メモ・チャット貼り付け等）**
- テキストから Issue として意味のある単位を抽出する
- 複数の課題が混在している場合はどの単位で Issue を作るかユーザーに確認する

---

## ステップ 2: 重複チェック（任意）

入力が十分に具体的な場合のみ `issue_list` を呼んで類似Issueがないか確認する。
完全に同じ内容があれば既存 Issue を案内して終了。
類似するものがあれば「#N と似ていますが新しく作りますか？」と確認する。

> タイトルのみの入力など曖昧な場合は省略してよい。

---

## ステップ 3: Issue を作成する

```
issue_create(
  title="<簡潔で具体的なタイトル>",
  body="<整形した本文>",
  status="draft"
)
```

### body のフォーマット

```markdown
## 目的・背景
<何のためにやるのか、なぜ必要か>

## やること
- <具体的なタスク>
- <...>

## 受け入れ条件
- <どうなればOKか>
- <...>
```

情報が少ない場合は空欄や TODO のままで作成してよい。完璧な Issue より「存在する Issue」の方が価値がある。

---

## ステップ 4: 結果を報告する

```
✅ Issue を作成しました

#<N> <タイトル>
ステータス: draft

詳細を追記したり、実装を始める場合は `jog-start` を使ってください。
```

---

## やってはいけないこと

- `status="ready"` にする（このスキルは draft 作成専用）
- Craft や worktree を作る（それは jog-start の仕事）
- 曖昧なまま複数 Issue を一度に大量作成する（ユーザーに確認を取る）
